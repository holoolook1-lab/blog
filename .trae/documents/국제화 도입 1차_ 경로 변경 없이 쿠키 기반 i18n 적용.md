## 목표
- 경로 재구성 없이 현재 구조에서 즉시 적용 가능한 국제화(i18n) 1차 도입
- 핵심 화면(Home/Posts/Header/Footer) 텍스트를 다국어(ko/en)로 전환하고 언어 토글 제공

## 범위
- 라이브러리: next-intl 추가
- 로케일: ko(기본), en(추가)
- 메시지: src/messages/ko.json, src/messages/en.json
- 프로바이더: 루트 레이아웃에서 NextIntlClientProvider로 쿠키 기반 메시지 주입
- 토글: Header에 LanguageSwitcher 추가(쿠키 설정 API 경유)
- 페이지 변환: Home, Posts 목록의 정적 텍스트를 i18n 키로 교체

## 구현 항목
1) 의존성 추가
- package.json에 "next-intl" 추가(설치 필요)

2) 메시지/로케일 유틸
- src/i18n/config.ts: 지원 로케일 및 기본값 정의
- src/i18n/getLocale.ts: 쿠키("locale")에서 현재 로케일 읽기
- src/messages/ko.json, src/messages/en.json: 공용/홈/포스트/버튼 등 핵심 키 정의

3) 프로바이더 주입
- src/app/layout.tsx: 서버에서 쿠키 로케일과 메시지를 로드해 NextIntlClientProvider로 children 감싸기

4) 언어 토글
- src/app/api/locale/route.ts: 쿠키("locale") 설정 API(204 응답)
- src/components/layout/LanguageSwitcher.tsx: 토글 UI(ko/en), 호출 후 페이지 새로고침
- Header에 스위처 삽입

5) 페이지 변환
- src/app/page.tsx: 서버에서 getTranslations로 문자열 교체
- src/app/(blog)/posts/page.tsx: 페이지 제목/버튼/빈 상태 문구 등을 i18n 키로 교체

## 이후 단계(선택)
